# FLARE Security Analysis

The canonical security analysis is **[`security.md`](security.md)** — a consolidated risk register with severity, status, code provenance, and test pointers for every observation made while building the prototype.

The per-phase files (`phase0-findings.md` … `phase4-findings.md`) are historical changelogs of how each finding was discovered and resolved during development. They are kept for provenance but a reviewer should read `security.md` first.
